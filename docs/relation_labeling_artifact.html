Module · nlp_relation.py

Labeling character relationships
with multilingual sentiment

How the project turns a co-occurrence pair into one of three edge colors — friendly, hostile, neutral — using cardiffnlp/twitter-xlm-roberta-base-sentiment over French prose.

Where it sits in the pipeline

NER→ Aliases→ Co-occurrence→ Relation labeling→ Graph→ HTML viz

Only AI-model component of an otherwise rule-based pipeline. Input: pairs of co-occurring characters per chapter. Output: {(charA, charB): "friendly" | "hostile" | "neutral"} dict that drives edge color in all_networks.html (green / red / gray).

Three labels, three colors

Friendly

pos − neg > 0.1. Edge rendered green.

Hostile

pos − neg < −0.1. Edge rendered red.

Neutral

Magnitude under 0.1, or zero snippets. Gray.

Step 1 — Pull co-occurrence snippets

For pair (charA, charB), find every text region where any alias of A sits within distance_max tokens of any alias of B. Cap at MAX_CONTEXTS_PER_PAIR = 5 per pair — sentiment classification is the bottleneck.

Why aliases, not just canonical names? Raw text contains "Empereur", not "Cléon". Co-occurrence detection works on canonical forms post-resolution, but here we re-scan source prose to grab exact spans.

Step 2 — Score each snippet

Run XLM-RoBERTa sentiment on each snippet. Model returns 3-class softmax: [neg, neu, pos]. Compute sentiment_score = pos − neg. 

for snippet in context_snippets:
    text = f"{charA} et {charB}: {snippet[:400]}"
    tokens = _tokenizer(text, return_tensors="pt", truncation=True)
    output = _model(**tokens)
    scores = softmax(output.logits.detach().numpy()[0])
    sentiment_score = scores[2] - scores[0]   # pos − neg

    if   sentiment_score >  0.1: votes.append(("friendly", abs(sentiment_score)))
    elif sentiment_score < -0.1: votes.append(("hostile",  abs(sentiment_score)))
    else:                       votes.append(("neutral",  abs(sentiment_score)))

Two non-obvious choices

Prefix f"{charA} et {charB}: ". Anchors model attention on the pair's interaction, not the scene's mood. Joyful festival where two enemies meet by chance must not read as friendly. Prefix forces judgment of the pair, not the prose.
±0.1 dead band. Captures model confidence. Strong positive [0.05, 0.10, 0.85] → score 0.80 → friendly. Weak signal [0.30, 0.40, 0.30] → score 0.00 → neutral. Avoids labeling on noise.

Step 3 — Weighted majority vote

Each snippet votes once. Vote weight = abs(sentiment_score) — confident snippets outweigh hesitant ones. Sum weights per label; max wins. 

totals = {"friendly": 0, "hostile": 0, "neutral": 0}
for label, weight in votes:
    totals[label] += weight
winner = max(totals, key=totals.get)

One confident hostile (weight 0.8) beats three weak neutrals (0.05 each). Counts of votes don't matter — total weight does.

Step 4 — Cache by (chapter, sorted pair)

Sentiment inference is the slow step. Cache key: (chapter_id, *sorted([charA, charB])).

  • Per chapter — same pair has different dynamics across the book (Cléon ↔ Demerzel evolves).
  • Sorted pair — collapses (A,B) and (B,A) to one entry.
  • Distance-invariant — cache stores final labels, not snippet scores. Changing distance_max does not require flushing.

Known biases

Issue Effect Mitigation
Hostile bias 239 hostile vs 58 friendly across 37 chapters. Dramatic French prose registers negative even for allies. Per-chapter calibration; raise threshold; switch to NLI with friendly prompts.
Single-snippet pairs One ambiguous snippet decides the label. Fragile. Drop to neutral when only 1 snippet and weight < X.
French-only prefix "et" hardcoded. Swap to "and" for English corpora.
Sarcasm / irony blind Friendly insults between allies read as hostile. No good fix at this model size.
